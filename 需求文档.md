# 待办事项应用需求文档

## 1. 项目概述
开发一个功能完整的待办事项应用，帮助用户高效管理日常任务，支持多种筛选和统计功能，数据持久化存储。

## 2. 功能需求

### 2.1 核心功能
- **添加待办事项**：
  - 标题（必填）
  - 描述（可选）
  - 截止日期（可选）
  - 优先级（高、中、低）
  - 分类（可自定义）

- **查看待办列表**：
  - 列表展示所有待办事项
  - 每个事项显示关键信息（标题、状态、截止日期、优先级、分类）
  - 支持分页或滚动加载

- **编辑待办事项**：
  - 修改已有事项的所有属性
  - 实时保存

- **删除待办事项**：
  - 单个删除
  - 批量删除

- **状态管理**：
  - 待办、进行中、已完成状态切换
  - 支持拖拽改变状态

### 2.2 高级功能
- **筛选功能**：
  - 按分类筛选
  - 按优先级筛选
  - 按状态筛选
  - 组合筛选

- **搜索功能**：
  - 支持按标题和描述搜索
  - 实时搜索结果
  - 高亮匹配内容

- **数据持久化**：
  - 刷新页面数据不丢失
  - 支持数据导出备份

- **统计功能**：
  - 完成率统计
  - 待办数量统计
  - 按分类统计
  - 按优先级统计
  - 数据可视化图表

## 3. 技术栈选择

### 3.1 前端技术栈
- **框架**：React 18
- **UI组件库**：Ant Design 5
- **状态管理**：React Hooks + Context API（或Zustand）
- **样式方案**：CSS Modules + Less
- **构建工具**：Vite
- **日期处理**：Day.js
- **图表库**：ECharts 或 Recharts

### 3.2 数据存储
- **本地存储**：localStorage（基础版）
- **数据库**：IndexedDB（进阶版，支持更大数据量）

### 3.3 开发工具
- **代码编辑器**：VS Code
- **版本控制**：Git
- **代码规范**：ESLint + Prettier

## 5. 数据模型设计

### 5.1 待办事项数据结构
```javascript
{
  id: string,              // 唯一标识
  title: string,           // 标题
  description: string,     // 描述
  dueDate: string,         // 截止日期（ISO格式）
  priority: string,        // 优先级：'high' | 'medium' | 'low'
  category: string,        // 分类
  status: string,          // 状态：'todo' | 'in-progress' | 'done'
  createdAt: string,       // 创建时间
  updatedAt: string        // 更新时间
}
```

### 5.2 分类数据结构
```javascript
{
  id: string,              // 唯一标识
  name: string,            // 分类名称
  color: string            // 分类颜色
}
```

## 6. 页面设计

### 6.1 首页（待办列表）
- 顶部：搜索栏 + 添加按钮
- 左侧：筛选面板（分类、优先级、状态）
- 中间：待办事项列表
- 右侧：统计卡片（完成率、待办数量等）

### 6.2 统计页面
- 顶部：时间范围选择
- 中间：统计图表（完成率趋势、分类分布、优先级分布）
- 底部：详细统计数据表格

### 6.3 设置页面
- 分类管理（添加、编辑、删除分类）
- 数据管理（导出、导入、清空）
- 主题设置（浅色/深色模式）
